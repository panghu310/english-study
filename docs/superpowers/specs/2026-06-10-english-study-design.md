# 英语小词典学习网页设计

## 背景

这个项目是一个自用英语学习网页，目标不是做复杂背单词系统，而是做成接近老式小词典的学习台。每个单词用音节拆分降低阅读和发音压力，并配一个极简中文释义。第一版先做小样本，验证在手机和平板上是否愿意长期使用。

最终词表方向是 Oxford 5000 加 IT 高频 1000 词，约 6000 词。第一版只放约 50 个样本词，包含日常高频词和开发/技术词。

## 第一版目标

- 做一个可本地开发、以后可部署到 GitHub Pages 的静态网页。
- 每个词显示音节拆分、简明中文释义和朗读按钮。
- 所有词默认进入“未掌握”分区。
- 用户手动把词移动到“需复习”或“已掌握”分区。
- 学习状态保存在当前浏览器本机，不做账号、不做同步、不做导入导出。
- 第一版用浏览器自带英文朗读，后续再升级本地音频包。

## 非目标

- 不做拼写测试。
- 不做造句练习。
- 不做自动复习算法。
- 不做账号系统。
- 不做 iPhone 和 iPad 之间的进度同步。
- 不在第一版生成 6000 词音频包。

## 页面结构

页面顶部显示产品名和统计信息：

```text
英语小词典
未掌握 50｜需复习 0｜已掌握 0
```

主体区域使用三个分区标签：

```text
未掌握｜需复习｜已掌握
```

默认进入“未掌握”分区。切换分区时，只显示当前分区内的单词。

## 单词呈现

每个单词用一行或紧凑卡片呈现：

```text
com.put.er      计算机      朗读
需复习  已掌握
```

单词数据包含：

- 原始单词：用于朗读，例如 `computer`
- 音节拆分：用于显示，例如 `com.put.er`
- 中文释义：极简释义，例如 `计算机`
- 分类标签：可选，例如 `daily` 或 `it`

## 分区规则

系统内置三个状态：

- `new`：未掌握
- `review`：需复习
- `known`：已掌握

所有单词初始状态都是 `new`。

在“未掌握”分区：

- 可以移到“需复习”
- 可以移到“已掌握”

在“需复习”分区：

- 可以移回“未掌握”
- 可以移到“已掌握”

在“已掌握”分区：

- 可以移回“需复习”
- 可以移回“未掌握”

这样可以避免误点后无法恢复。

## 朗读设计

第一版使用浏览器的 `SpeechSynthesis` 朗读英文单词。

朗读按钮逻辑：

1. 用户点击某个单词的朗读按钮。
2. 页面读取原始单词，而不是读取音节拆分文本。
3. 如果浏览器支持英文朗读，则播放该单词。
4. 如果朗读不可用，则显示短提示。

后续升级本地音频包时，保持同一个按钮交互：

1. 优先播放本地音频文件。
2. 如果本地音频不存在，再使用浏览器朗读兜底。

## 保存设计

第一版使用 `localStorage` 保存状态。

保存内容只包括单词状态映射，例如：

```json
{
  "computer": "known",
  "database": "review"
}
```

刷新页面后恢复进度。清除浏览器数据、换设备或换浏览器时，进度可能丢失；第一版接受这个限制。

## 项目结构

第一版使用极简静态网页结构：

```text
index.html
src/
  words.js
  app.js
  styles.css
```

各文件职责：

- `index.html`：页面骨架，引入脚本和样式。
- `src/words.js`：小样本词表数据。
- `src/app.js`：分区渲染、状态保存、朗读逻辑。
- `src/styles.css`：适配手机和平板的视觉样式。

这个结构可以直接本地开发，也方便后续放到 GitHub Pages。

## 视觉和交互原则

- 页面以手机 Safari 使用为主，按钮和点击区域要足够大。
- 单词本身要突出，释义保持短。
- 音节拆分使用等宽或接近等宽字体，便于识别点号分隔。
- 三个分区切换要清楚，当前分区有明显状态。
- 不使用复杂动画，不做花哨首页，打开后直接进入学习。

## 错误处理

- 朗读不可用时，显示短提示，不阻塞其它功能。
- 本地保存失败时，页面仍可使用，但提示进度可能无法保存。
- 当前分区没有单词时，显示简短空状态。

## 测试和验证

第一版完成后需要验证：

- 本地开发页面可以正常打开。
- 三个分区切换正常。
- 单词可以在三个分区之间移动。
- 刷新页面后分区状态仍然存在。
- 点击朗读按钮能读取原始英文单词。
- 在窄屏尺寸下文字和按钮不重叠。
- 未来部署到 GitHub Pages 时不依赖后端服务。

## 后续升级

第一版确认好用后，再考虑：

- 扩展到完整 Oxford 5000 加 IT 高频 1000 词。
- 增加本地音频包。
- 增加导出/导入进度。
- 增加词表筛选，例如日常词、技术词。
- 增加简单搜索。
